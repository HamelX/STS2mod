# Gunslinger Arcane Canon (Pass 1)

Pass-1 목적: 기존 실린더(로드/격발/회전) 코어를 유지한 채, 용어와 텍스트 톤을 **아케인 건슬링어/운명탄 의식** 중심으로 통일한다.

## 1) 정식 용어 매핑 (Canon)

- **Cylinder** → **실린더** (시스템 코어 자원판)
- **Chamber** → **약실** (실린더의 개별 슬롯)
- **Firing chamber** → **격발 약실** (12시 기준 현재 발사 슬롯)
- **Trigger pull** → **방아쇠 당김 / 격발 시도**
- **Ammo lanes (expandable concept)** → **탄도**
  - 현재 구현은 단일 원형 실린더이지만, 문서/텍스트에서 “탄도”를 사용해 추후 확장 레인 설계를 수용
- **Imprint** → **각인**
- **Ricochet** → **반향** (기존 키워드 [도탄]은 기계적 호환 위해 유지)
- **Manifest: Ricochet** → **현현: 반향**
- **Tracer rhythm flag** → **추적 박자**

## 2) 코어 루프 문장 (UI/문서 공통)

> 장전으로 실린더를 세팅하고, 방아쇠를 당겨 격발하며, 빈 약실도 회전 상태로 활용한다.

축약:
1. 장전(탄종/봉인 레벨 준비)
2. 격발(성공 시 탄환 소모 + 효과 발생)
3. 회전(실패 격발 포함)
4. 상태/표식(각인·반향·봉인 레벨) 운영

## 3) 네이밍 원칙 (Pass-2까지 유지)

1. “사격” 계열 카드명은 가능하면 **격발** 계열로 통일
2. 탄약 설명은 “탄환” + “탄도” 병행 가능 (UI 호버는 탄도 우선)
3. 기계 키워드는 즉시 교체하지 않는다
   - 예: [도탄] 토큰은 현재 시스템 안정성 때문에 유지
4. 이름이 발포/사격을 암시하면 반드시 **trigger-pull 파이프라인**(TryConsumeCurrent + AdvanceChamber)과 합치되어야 함

## 4) Pass-2 예고 (메커닉 정렬)

- [도탄] ↔ [반향] 표시 체계 통합 여부 결정
- 탄도(레인) 확장 시 UI/파워 저장구조 영향 분석
- 카드별 “격발 실패(빈 약실)” 보상/페널티 정렬
- 탄종별 비주얼/툴팁 색채 문법 통일
